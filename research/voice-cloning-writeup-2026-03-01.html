Cloning an Arabic Teacher's Voice

01

The Motivation

Alif, my Arabic learning app, uses ElevenLabs for text-to-speech. Every sentence the learner reviews can be read aloud — the Arabic script lights up, the audio plays, and you get both the reading and the listening in one pass. It's probably the feature I use most.

The problem is the voice. ElevenLabs' stock Arabic voices are fine — technically correct, clearly enunciated — but they sound like a GPS navigation system reading the Quran. There's no warmth, no teaching instinct, no sense that someone is speaking to you. The best language teachers have a particular cadence: they slow down on new vocabulary, they pause at clause boundaries, they repeat key phrases with slight variation. Stock TTS voices don't do this.

I've been watching a TikToker called RootsOfKnowledge who does exactly this. His "Slow Arabic" series is remarkable — deliberate, clear MSA (Modern Standard Arabic) with a natural teacher's rhythm. He speaks the way you'd want a tutor sitting across from you to speak: unhurried, warm, and precise. 25 years of teaching Arabic distilled into three-minute videos.

The idea Clone his voice using ElevenLabs, then use it to generate TTS for every sentence in the app. The app already supports custom voice IDs — it would be a one-line config change to switch voices. The question was whether the pipeline from TikTok → audio extraction → voice cloning was actually feasible.

03

The TikTok Wall

The plan seemed simple: use yt-dlp to download all his TikTok videos, extract the audio, upload to ElevenLabs. Twenty minutes of work, tops.

It took considerably longer than twenty minutes.

Attempt 1: yt-dlp direct

yt-dlp -x --audio-format mp3 "https://www.tiktok.com/@roots_of_knowledge"

Immediate failure. Read timed out. TikTok's servers simply refused to respond to yt-dlp's requests. The error message hinted at the problem: "No supported JavaScript runtime could be found" and "This user's account is either private or has embedding disabled." The account isn't private — TikTok just doesn't serve content to anything that doesn't look like a real browser.

Attempt 2: Browser impersonation

yt-dlp supports the --impersonate chrome flag, which spoofs a full browser TLS fingerprint. This requires the curl_cffi library — which wasn't installed. After installing it: 

yt-dlp --impersonate chrome "https://www.tiktok.com/@roots_of_knowledge/video/..."

Exit code 134. A segfault — or something equally catastrophic deep in the impersonation layer. The process just died silently mid-download.

Attempt 3: Browser cookies

yt-dlp --impersonate chrome --cookies-from-browser chrome "..."

This hung at "Extracting cookies from chrome" indefinitely. Presumably Chrome's cookie store was locked, or the cookie format had changed. Killed after two minutes.

Attempt 4: Third-party APIs

Pivoted to tikwm.com, a popular TikTok scraping API. The user listing endpoint returned 403 Forbidden. But interestingly, the individual video download endpoint worked — when given a specific video URL, it could return the video without watermark.

The problem: I only knew three video URLs from web search results. I needed to enumerate all his videos first.

Attempt 5: Browser automation

Launched a headless browser with agent-browser and navigated to his TikTok profile. The profile loaded — I could see his bio, follower count, playlists — but the video grid showed "Something went wrong. Sorry about that! Please try again later." TikTok detected the automated browser and refused to render the actual content.

TikTok's defenses in 2026 TikTok has implemented multiple layers of bot detection: TLS fingerprinting (blocks non-browser HTTP clients), CAPTCHA challenges (slider puzzles on profile pages), JavaScript rendering gates (video grid requires client-side JS execution with valid session), and behavioral analysis (automated scrolling patterns trigger content blocking). Each layer independently blocks a different class of scraping tool. Defeating all of them simultaneously is a serious engineering project.

After five failed approaches, the total video haul from TikTok was: 3 videos, downloaded via the tikwm.com single-video endpoint using URLs found through web search. About 90 seconds of audio. Not enough.

04

The YouTube Pivot

The browser screenshot of his TikTok profile revealed something useful: his bio includes a YouTube link — youtube.com/@rootsofknowledge. YouTube is vastly more scraping-friendly than TikTok. One command: 

yt-dlp --flat-playlist --print "%(id)s | %(title)s | %(duration)s" \
  "https://www.youtube.com/@rootsofknowledge/videos"

1,989 videos. Listed in under 30 seconds. No impersonation, no cookies, no CAPTCHA. YouTube still works.

05

The ElevenLabs Upload

ElevenLabs offers two flavors of voice cloning: Instant Voice Clone (IVC) and Professional Voice Clone (PVC). IVC takes up to 45 minutes of audio, processes in seconds, and produces a usable voice immediately. PVC takes 30+ minutes of audio but trains a more accurate model over several hours. For a first test, IVC was the obvious choice.

The API key saga

The upload script was straightforward — multipart POST to /v1/voices/add with the audio files. But the first attempt failed: 

// Response from ElevenLabs API
{
  "status": "missing_permissions",
  "message": "The API key you used is missing the permission
              create_instant_voice_clone to execute this operation."
}

The existing API key — the one already used in Alif for TTS generation — was scoped to only text-to-speech. It couldn't create voices. ElevenLabs API keys have granular permissions, and the original key was deliberately minimal.

After updating the key permissions (which didn't seem to take effect — the API still rejected it), a completely new key was generated. This one worked, but came with its own surprise: the new key had a 10-credit quota by default, far too small to generate any speech. The solution was using the new key for voice creation and the old key for TTS generation. Two keys, two purposes.

The 45-minute limit

The first upload attempt with the new key included all 13 shortest files — totaling about 48 minutes. ElevenLabs rejected it: 

{
  "status": "voice_too_long",
  "message": "Voice samples must be at most 45 minutes long."
}

Trimmed to 11 files (~40 minutes) and tried again: 

// The upload that worked
POST /v1/voices/add
Content-Type: multipart/form-data

name: "RootsOfKnowledge Arabic"
description: "Arabic teacher voice..."
labels: {language: "Arabic", accent: "MSA", gender: "male"}
remove_background_noise: true
files: [11 MP3 files, 44.6 MB total]

The Result

The voice was available instantly after the upload. Testing it with the original TTS API key (which has the right quota):

مرحباً، أنا أتعلم اللغة العربية. هذا صوت جميل جداً. "Hello, I am learning the Arabic language. This is a very beautiful voice."

4.2 seconds of audio. 66KB MP3. Generated in under 2 seconds using eleven_multilingual_v2 with the cloned voice. The voice is recognizably him — the warm baritone, the measured pacing, the slight breathiness between phrases. It's not perfect; the cloned voice is slightly smoother than his natural speech, losing some of the micro-hesitations that make a real teacher sound human. But for sentence-level TTS in a learning app, it's remarkably good.

07

What I Learned

Lesson 1: Start with YouTube If your target creator has both TikTok and YouTube, skip TikTok entirely. YouTube's scraping ecosystem is mature and functional. TikTok's is a constantly shifting arms race. The audio quality from YouTube is also higher — YouTube serves 128-192kbps audio vs TikTok's compressed mobile format.
Lesson 2: Slow Arabic content is ideal for cloning Language teaching content is nearly perfect voice cloning material: solo speaker, clear enunciation, minimal background noise, consistent recording environment, varied vocabulary and intonation. "Slow Arabic" is even better — the deliberate pacing means less overlapping phonemes and cleaner samples for the model to learn from.
Lesson 3: ElevenLabs API keys don't hot-reload permissions Editing an existing API key's permissions doesn't seem to take effect immediately — or possibly at all. Creating a new key with the right permissions from the start is the reliable path. Keep separate keys for separate concerns (creation vs. generation).
Lesson 4: IVC vs PVC Instant Voice Clone was good enough for this use case. Professional Voice Clone would produce a higher-fidelity voice but takes hours to train and requires the Creator+ plan. For a private learning app where the voice is one component among many, IVC's "good enough in seconds" beats PVC's "better in hours." If the voice becomes central to the product experience, upgrading to PVC is a natural next step — I already have 157 minutes of clean audio ready.
Lesson 5: 40 minutes is more than enough ElevenLabs' IVC limit is 45 minutes, but even 10-15 minutes of clean speech produces a recognizable clone. The marginal quality improvement from 15 → 40 minutes is modest. The most important thing is sample diversity: different intonations, different speeds, different emotional registers. Ten minutes of varied speech beats forty minutes of monotone.
08

What's Next

The voice exists. It works. It's not integrated into Alif yet — that's a deliberate choice. Before swapping the voice, I want to:

A/B test the voice quality. Generate 20-30 sentences with both the current stock voice (Chaouki, eleven_multilingual_v2) and the cloned voice, then listen to both sets. The cloned voice should be better for Arabic-specific phonetics — it was trained on Arabic, not on multilingual data — but stock voices sometimes handle edge cases (numbers, transliterated words, code-switching) more gracefully.

Try Professional Voice Clone. I have 157 minutes of audio sitting in /private/tmp/claude/roots_of_knowledge/audio/. PVC needs 30+ minutes; I have 5x that. The Creator plan supports one PVC slot. Worth trying to see if the fidelity improvement is perceptible for short sentence-level clips.

Tune the voice settings. The current settings (stability 0.85, similarity 0.75) were copied from the existing Chaouki voice. The cloned voice might perform better with different parameters — possibly lower stability for more natural variation, or higher similarity for stronger voice character on very short phrases.

Consider open-source alternatives. Coqui XTTS and Fish Speech both support Arabic and can run locally with zero per-request cost. For a single-user app generating thousands of sentences, the ElevenLabs per-character cost adds up. A local model with unlimited generation could be more sustainable long-term — if the quality is comparable.
