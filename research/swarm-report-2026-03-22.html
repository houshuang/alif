Dynamic Session Sizing

sh/dynamic-session-sizing
PR #22

Session length now adapts to your 2-day accuracy instead of being fixed at 10 cards. At 96% accuracy, you get 18 cards per session — nearly double the practice volume.

  • _dynamic_session_limit() computes limit from 2-day ReviewLog accuracy
  • Thresholds: <90% → 10, ≥90% → 14, ≥95% → 18 cards
  • Extracted _get_2day_accuracy() as shared helper, also used by intro slot computation
  • Frontend removed hardcoded limit=10 from all 4 API call sites
  • Explicit limit parameter still overrides (for tests, simulations)
Learning impact: At 96% accuracy with 10-card sessions, you complete in ~3 minutes. 18 cards means more encounters per session, more collateral learning, and the natural difficulty increase pushes toward the 85% optimal accuracy zone (Wilson et al. 2019).

Comprehensibility Gate Relaxation

sh/familiar-encountered-gate
PR #25

Words you've seen 8+ times in stories/sentences but never formally studied now count as "known" for the comprehensibility threshold. This allows harder, more natural sentences.

  • FAMILIAR_ENCOUNTER_THRESHOLD = 8 — tunable constant
  • Applied to both comprehensibility gates (candidate filtering + greedy selection)
  • Unknown scaffold cap also benefits — familiar words no longer count as "unknown"
  • Full gate audit completed per CLAUDE.md rule 8
Learning impact: Creates a virtuous cycle — harder sentences contain more encountered words as scaffold → more collateral introductions → those words become familiar → even harder sentences become eligible. Especially powerful for story readers who've accumulated many encountered words.

Confusable Pair Exclusion

sh/confusable-pair-exclusion
PR #24

Arabic words that share a dotless skeleton (rasm) — like بنت "girl" and بيت "house" — are now prevented from appearing in the same session. Reduces interference between visually similar words.

  • compute_rasm() strips dots: بنت → ٮٮٮ, بيت → ٮٮٮ → confusable
  • Fixed missing mapping for standard Arabic yeh (ي U+064A) — was only mapping Farsi yeh
  • build_confusable_index() — single DB query + dict grouping
  • Session builder drops lower-priority confusable sentences; they appear next session
Learning impact: Arabic's dot system was a late historical addition — the original rasm was dotless. Many letter pairs are identical without dots (ب/ت/ث/ن/ي are all ٮ). Presenting confusable pairs in the same session creates destructive interference. Research (Carvalho & Goldstone 2014) confirms spacing similar items reduces confusion.

Response Time Difficulty Signal

sh/response-time-signal
PR #23

Words you rate "Good" but take 8 seconds to recognize now get priority — response_ms was already logged but never used. Catches weak automaticity that FSRS ratings miss.

  • fluency_service.py computes per-word fluency as global_median / word_median
  • Batch computation via 2 SQL queries — precomputed once per session build
  • Words with fluency < 0.7 get +0.3 additive priority boost
  • Only FSRS words (not acquiring) to avoid double-boosting
  • Fluency score exposed in word detail endpoint
Learning impact: FSRS only sees explicit ratings. A word rated "Good" in 8 seconds vs 1.5 seconds gets identical scheduling. But the slow word has weak automaticity — you can decode it but it's not fluent. The fluency signal surfaces these words for more practice without changing FSRS state.

Mnemonic Regeneration

sh/mnemonic-regeneration
PR #21

Words that keep failing despite having memory hooks get fresh, different mnemonics. The old hook is preserved for audit, and a "previous didn't stick" prompt guides the LLM toward a new approach.

  • find_stuck_hook_words() — ≥4 reviews, <50% recent accuracy, existing hooks
  • check_and_regenerate_stuck_hooks(db) — background task, 3 words/run, Claude Haiku
  • 7-day cooldown between regenerations, regeneration_count tracked
  • Old hooks preserved in previous_hooks field
  • CLI script: regenerate_stuck_mnemonics.py --dry-run --limit N
Learning impact: SMART research (Balepur, EMNLP 2024) found that what learners think helps and what actually helps disagree (0.4-0.5 agreement). A mnemonic that seemed clever may not resonate with your specific associative memory. Regenerating with a different keyword/approach gives the word a second chance.

Variant Verification Audit

sh/variant-audit
PR #20 · merged

Detects semantically-distinct words incorrectly merged as variants (e.g., صيادية "dish" merged into صياد "hunter"). Checks gloss coherence, then LLM-verifies suspicious pairs.

  • compute_jaccard_similarity() — tokenized gloss overlap scoring
  • verify_variant_pair() — import-time check for new variant assignments
  • audit_variants.py script with --fix and --threshold flags
  • LLM verification via Claude Haiku with Gemini fallback
  • 17 tests across 3 test classes
Learning impact: Wrongly merged variants mean reviews credit the wrong word. If صيادية (a fish dish) is merged into صياد (hunter), every time you review a sentence about fishing, your SRS state for the dish gets updated. This corrupts FSRS scheduling for both words silently.

04 Research Backing

Evidence Base

  • Wilson et al. 2019 (Nature) — Optimal training accuracy ~85%. Dynamic sessions push toward this by increasing volume for strong learners.
  • Carvalho & Goldstone 2014 — Spacing similar items reduces confusion. Confusable pair exclusion implements this directly.
  • Uchihara 2019 meta-analysis — 8-12 meaningful encounters needed for stable vocabulary. Familiar gate uses 8 encounters as the "practically known" threshold.
  • Balepur, EMNLP 2024 (SMART) — Learner preferences and actual effectiveness disagree. Mnemonic regeneration doesn't ask the learner — it measures outcomes.
  • Nakata 2017 — Within-session retrieval sweet spot is ~3. Response time signal catches words that need more retrievals despite seeming "known."
