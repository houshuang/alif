# Backend Services

All services in `backend/app/services/`.

## Scheduling & Review
- `fsrs_service.py` ‚Äî FSRS spaced repetition. Snapshots pre-review state in fsrs_log_json for undo. Safety-net auto-create for unknown ULK (normally handled by acquisition path in sentence_review_service). Triggers `regenerate_memory_hooks_premium` on fresh lapse (background thread).
- `sentence_selector.py` ‚Äî Session assembly: greedy set cover, comprehension-aware recency (understood=1d/partial=4h/no_idea=30min), difficulty matching, easy-bookend ordering. Focus cohort filtering (MAX_COHORT_SIZE=2000). **Reserved-slot auto-introduction**: reserves `INTRO_RESERVE_FRACTION` (30%) of session slots for new words during the aggressive 30/day trial when accuracy allows, even when due queue exceeds limit. Reserved auto-intro stops at `DAILY_AUTO_INTRO_TARGET=30`; at ‚â•90% recent accuracy the acquiring backlog cap is temporarily `HIGH_ACCURACY_INTRO_BACKLOG_CAP=120`. Also fires when session is undersized (backward-compat). `_intro_slots_for_accuracy()` maps recent accuracy to graduated rate (<70%‚Üí0, 70-85%‚Üí3, ‚â•85%‚Üí5 slots). Per-call cap: MAX_AUTO_INTRO_PER_SESSION=5. **Low-tier gate** (2026-04-13): when box-1 acquiring count > `LOW_TIER_BLOCK_BACKLOG` (60), candidates whose source is in `LOW_TIER_INTRO_SOURCES` (wiktionary, story_import, manual, flag_autocreate, unsourced) are filtered out ‚Äî even during undersized-session fill. Forces the learner to clear actively-encountered backlog (textbook_scan, book, active stories) before introducing words from passive frequency lists. **Fill phase**: when session is still undersized after main assembly + on-demand generation, a second auto-introduce pass runs. Within-session repetition for acquiring words with per-box targets (`BOX1_MIN_EXPOSURES`=4 for encoding, `BOX2_MIN_EXPOSURES`=2 for consolidation; multi-pass expanding intervals, `MAX_ACQUISITION_EXTRA_SLOTS`=15). Box-2 was lowered from 4 ‚Üí 2 on 2026-05-04 ‚Äî a single correct rating in box-2 already triggers Tier 1 graduation, so trailing reps were wasted on already-mastered words. Frontend additionally auto-skips a sentence card whose primary lemma was already answered correctly earlier in the same session (failed reviews keep the full re-teaching loop). **Within-session scaffold diversity**: tracks scaffold words across greedy set cover iterations, applies `SESSION_SCAFFOLD_DECAY` (0.5) exponential penalty per reuse ‚Äî prevents the same scaffold words from dominating every sentence. Comprehensibility gate (‚â•60% known scaffold words; acquiring box-1 excluded, encountered excluded ‚Äî only actively studied words count as known). **Unknown scaffold cap**: `MAX_UNKNOWN_SCAFFOLD` (2) ‚Äî sentences with >2 unknown non-target words rejected to prevent overwhelming density after large OCR batches; fill-phase pregenerated selection now applies the same cap. **Near-duplicate veto**: session selection rejects both high lemma-set Jaccard and near-identical normalized Arabic text; the same veto now applies in the pregenerated fill path. On-demand sentence generation: multi-target first (groups of 2-4), single-target fallback, parallelized via ThreadPoolExecutor (max 8 workers). Graceful degradation: if on-demand generation fails (DB locked, LLM error), session builder returns existing sentences instead of 500ing. No word-only fallbacks. **Variant‚Üícanonical resolution**: sentences with variant forms correctly cover canonical due words; `effective_id` used for scheduling only, `WordMeta.lemma_id` uses original `sw.lemma_id` for display/lookup, and response words include `canonical_lemma_id` so frontend intro-card interleaving can match variant surfaces to canonical cards. **Intro-card eligibility**: after all selection/fill phases, the session scans every non-function word in returned items, resolves canonical IDs, promotes cold `new`/`encountered` session words into acquisition, and builds first-time intro cards for all unseen session words. Rescue cards remain dynamically capped. **Book sentence preference**: 1.3x source_bonus for `source="book"` sentences over LLM-generated. `compute_sentence_diversity_score()` logs per-sentence metrics (scaffold_uniqueness, scaffold_freshness) for monitoring. **Never-reviewed boost**: acquiring words with `times_seen == 0` get `NEVER_REVIEWED_BOOST` (5.0x) score multiplier so their single-target sentences compete against multi-word FSRS sentences in greedy selection. **Overdue escalation** (2026-04-11): words >3 days overdue get a growing score multiplier (linear ramp up to `OVERDUE_ESCALATION_MAX` 4.0x at 17+ days) via `_overdue_escalation()`. Prevents acquisition and FSRS words from being starved by multi-word sentence dominance in the greedy scorer. **Selection transparency**: each `SentenceReviewItem` includes `selection_info` dict with `reason` (greedy_cover/acquisition_repeat/on_demand/fill_intro), `score`, `order`, `word_reason` (human-readable primary word state), and `components` (per-factor score breakdown: due_coverage, difficulty_match, grammar_fit, diversity, freshness, source_bonus, session_diversity, rescue, never_reviewed_boost, overdue_boost). **Fast mode performance**: when `skip_on_demand=True`, auto-introduction skips material generation (`skip_material_gen=True`) and lemma backfill uses fast dictionary lookup only (no CAMeL disambiguation) ‚Äî reduces session build from ~18s to ~1.2s. **Fill phase always runs**: even in fast mode, the fill phase fires when session is undersized. Uses `_find_pregenerated_sentences_for_words()` (fast DB queries, no LLM) to find existing active sentences for newly introduced words. `mode` parameter ("reading"/"listening") controls which comprehension column is used for recency filtering. **No LLM in session build**: mapping verification is NOT done during session build (would add 15-30s latency). All verification happens at generation time or in `warm_sentence_cache` Phase 4 (background). Unverified sentences may be shown if Phase 4 hasn't caught up yet, but they already passed generation-time verification. **Graduated tashkeel fading** (2026-03-27): scaffold words (`is_due=False`) fade at `min(tashkeel_threshold/3, 30d)`; target/due words fade at the full configured threshold (90d). Scaffold words at 30d+ stability don't need the crutch; due words still do.
- `sentence_review_service.py` ‚Äî Reviews ALL words equally. Routes acquiring‚Üíacquisition, skips encountered. **Collateral credit**: unknown words auto-introduced into acquisition (source="collateral") instead of straight to FSRS. **Variant‚Üícanonical redirect**: reviews of variant words credit the canonical lemma, with surface forms tracked in variant_stats_json. credit_type is metadata only. Post-review leech check runs on every rated word (sliding-window criterion ‚Äî a rating-3 review can still push the window below 50% by evicting an older correct one; fixed 2026-04-21). Undo restores pre-review state from snapshots.
- `acquisition_service.py` ‚Äî Leitner 3-box (4h‚Üí1d‚Üí3d). **Two-phase advancement**: box 1‚Üí2 always allowed (encoding), box 2‚Üí3 and graduation require `acquisition_next_due <= now` (consolidation). **Tiered graduation** (2026-03-03): Tier 0: first correct review ‚Üí instant graduation; Tier 1: 100% accuracy + 3 reviews ‚Üí any box; Tier 2: ‚â•80% accuracy + 4 reviews + box‚â•2; Tier 3 (standard): box‚â•3 + times_seen‚â•5 + accuracy‚â•60% + reviews on ‚â•2 UTC calendar days. `start_acquisition()` accepts `due_immediately=True` for auto-introduced words. **Preserves existing meaningful source** (book, duolingo, textbook_scan, etc.) ‚Äî only overwrites generic "study"/"encountered" sources. Triggers `regenerate_memory_hooks_premium` on box demotion from 2+ ‚Üí 1 (background thread). **A/B experiment**: randomly assigns `experiment_group` (intro_ab_card or intro_ab_sentence) on first acquisition start.
- `cohort_service.py` ‚Äî Focus cohort: MAX_COHORT_SIZE=2000 (raised from 200 on 2026-04-11). Acquiring words always included, rest filled by lowest-stability due words.
- `frequency_lanes.py` ‚Äî Main/slow lane classifier for the 30/day frequency-core experiment. Main lane includes all acquiring words, all frequency-core/proxy rank <=5,000 due words, and non-artifact due words. Slow lane samples low/null-rank artifact FSRS debt from book/OCR/story/scaffold paths at 10% of session budget. Shared by stats and session selection so the daily goal matches what the selector can serve.
- `leech_service.py` ‚Äî Auto-manage failing words. Detection: **sliding window** of last `LEECH_WINDOW_SIZE` (8) reviews with accuracy<50% AND total times_seen‚â•5. Sliding window prevents the "leech escape trap" where cumulative stats made it mathematically impossible for reintroduced words to improve. **Graduated cooldown**: 3d (1st) ‚Üí 7d (2nd) ‚Üí 14d (3rd+) via `leech_count` on ULK. Stats preserved on reintroduction (not zeroed). Fresh sentences generated and memory hooks ensured on reintro.

## Word Selection & Introduction
- `word_selector.py` ‚Äî Next-word algorithm with **strict priority tiers**: frequency-core top 500 (+240), frequency-core top 1000 (+210), active book (`200 - page*2.0`), frequency-core top 2000 (+170), frequency-core top 5000 (+120), active stories (+10), OCR/textbook_scan (+8), duolingo (+6), avp_a1 (+4), wiktionary/other (+0, strictly by freq). Within-tier tiebreakers: 40% freq + 30% root + 20% recency + 10% grammar + encountered bonus + category penalty. Topic acts as tiebreaker within OCR/Duolingo only (+0.3). Root-sibling interference guard. Excludes wiktionary refs and variant lemmas. **Frequency core**: `frequency_core_entries` is the primary general-reading curriculum; `select_next_words()` returns `frequency_core_rank`, and `_auto_introduce_words` records source `frequency_core` for those introductions. **Variant‚Üícanonical resolution**: `_resolve_to_canonical()` follows multi-hop variant chains (A‚ÜíB‚ÜíC resolves to C). `_active_story_lemma_ids()` and `_book_page_numbers()` resolve variant StoryWord lemma IDs to root canonicals, ensuring book page priority and story priority bonuses reach the actual introduction candidates. `select_next_words()` returns `story_id` for book/story candidates, `pattern_examples` (up to 4 words with same wazn from touched roots), `forms_translit` (ALA-LC transliterations for each form), and `root_family` (all root siblings with `wazn`/`wazn_meaning` per sibling ‚Äî used for the verb form derivation bridge). `get_root_family()` now includes `wazn`/`wazn_meaning` on each sibling entry. introduce_word() calls start_acquisition(), accepts `due_immediately` param for auto-introduction during sessions. **Auto-introduction source attribution**: `_auto_introduce_words` derives ULK source from priority tier (book‚Üí"book", frequency_core‚Üí"frequency_core", active_story‚Üí"story_import", textbook_scan/duolingo‚Üías-is, other‚Üí"auto_intro") and sets `Lemma.source_story_id` for book/story words. Never picks proper_name, onomatopoeia, or function words.
- `topic_service.py` ‚Äî Topical learning cycles. 20 domains, MAX_TOPIC_BATCH=15, MIN_TOPIC_WORDS=5. Auto-advance when exhausted/depleted.

## Sentence Generation & Validation
- `sentence_generator.py` ‚Äî LLM generation with 7-attempt retry loop, diversity weighting, full diacritics. Feeds validation failures back as retry feedback. Post-validation Claude Haiku quality review gate (naturalness + translation accuracy). Accepts `model_override` param (cron uses `claude_sonnet`, on-demand uses `gemini`). Multi-target generation: `generate_validated_sentences_multi_target()` + demand-weighted `group_words_for_multi_target()` for root-diverse 2-3 word groups; validator now requires at least two target hits for multi-target acceptances.
- `sentence_validator.py` ‚Äî Rule-based: tokenize ‚Üí strip diacritics ‚Üí strip clitics ‚Üí match known forms. `FUNCTION_WORDS` populated from `FUNCTION_WORD_GLOSSES` keys (~85 particles/prepositions/pronouns/conjunctions) ‚Äî excluded from story/book word counts, book page introduction, FSRS review credit, and scaffold diversity. `FUNCTION_WORD_GLOSSES` provides register-aware fallback glosses (e.g. ÿ´ŸÖ "then (after delay)", ÿ®ŸêŸÄ "in/by/with", ŸÇÿØ "indeed/may/already"). `FUNCTION_WORD_FORMS` kept for clitic analysis prevention. Public API: lookup_lemma(), resolve_existing_lemma(), build_lemma_lookup(), build_comprehensive_lemma_lookup() (ALL lemmas for sentence_word mapping). `validate_sentence_multi_target()` for multi-word validation. **CAMeL disambiguation**: `lookup_lemma()` calls `_camel_disambiguate()` (via `find_best_db_match()`) for ambiguous clitic-stripped matches and as last-resort fallback. Al-prefix length guard: stems < 3 chars skip al-prepend. **Three-pass lookup**: `build_lemma_lookup()` registers direct bare forms (pass 1), forms_json derived forms (pass 2), then verb conjugation forms (pass 3). Pass 3 generates ~33 conjugations per verb via `_generate_verb_conjugations()` ‚Äî past suffixes (9 person/number/gender) + present prefix/suffix combinations (4 prefixes √ó 6 suffix variants) for all verbs with `present` in forms_json. Works well for sound verbs; partial coverage for weak verbs. Indexes extended forms_json keys (past_3fs, past_3p, imperative, passive_participle). **Tanwin-alif stripping**: `strip_tanwin_alif()` removes trailing alif seat of fathatan (ÿ≥ÿπŸäÿØÿß‚Üíÿ≥ÿπŸäÿØ) ‚Äî applied to both scaffold and target word matching. **LLM mapping verification**: `verify_and_correct_mappings_llm()` sends word-lemma pairs for contextual verification ‚Äî tries Gemini Flash ‚Üí Claude Haiku fallback; returns `None` on total failure (callers must discard/skip sentence). `correct_mapping()` finds correct lemma in DB only (never auto-creates). `apply_corrections()` is the shared entry point for all 7 correction sites ‚Äî handles the 3-way result check (corrected / not found / same lemma) and logging; returns failed positions so callers decide fate. Also returns per-position `failure_reasons` via the log entry (`"same_lemma"` = homograph vocab-gap, strong signal; `"not_found"` = LLM proposed lemma not in DB) ‚Äî consumed by `scripts/missing_lemma_candidates.py` for ranked curation into `import_scaffold_lemmas.py`. Active in production (`VERIFY_MAPPINGS_LLM=1`).
- `pipeline_tiers.py` ‚Äî Due-date tiered sentence allocation. Classifies words into 4 urgency tiers: Tier 1 (due ‚â§12h, target 3, floor 2), Tier 2 (12-36h, target 2, floor 1), Tier 3 (36-72h, target 1, floor 0), Tier 4 (72h+, target 0, actively retired). Used by `update_material.py`, `material_generator.py`, and `rotate_stale_sentences.py`. Pool size bounded by review urgency (~200 tier 1-3 words), not vocabulary size.
- `sentence_self_correct.py` ‚Äî Self-correcting tool-enabled Sonnet session (shipped 2026-04-20). `generate_sentences_self_correct_batch(target_lemma_ids, db_path, needed_per_target=2)` opens ONE Claude CLI session with `Bash,Read` tools, writes `vocab_prompt.txt` + `vocab_lookup.tsv` + `targets.json` + `validator.py` into a work_dir, then lets Sonnet draft ‚Üí `python3 validator.py "<arabic>" "<target_bare>"` ‚Üí surgically swap unknown words ‚Üí re-validate until `needed_per_target` sentences exist per target. System prompt includes rules A-E against unanchored 3rd-person verbs, bare definite subjects, and forced combinations that produced the 2026-04-19 single-target 67%‚Üí95% quality jump once tightened. Used by `material_generator.py` as the default generation path. Schema: `{results: [{target_lemma_id, sentences: [{arabic, english, transliteration, edits}]}]}`. Single-target variant `generate_sentences_self_correct()` also exists but the batch wrapper is the production path.
- `material_generator.py` ‚Äî Orchestrates sentence + audio generation for a word. Default path (2026-04-20): `generate_material_for_word()` delegates to `batch_generate_material()`, which opens ONE tool-enabled Sonnet CLI session per batch via `sentence_self_correct.generate_sentences_self_correct_batch()` (N up to ~10 lemmas is the sweet spot ‚Äî N=15 regressed to 56% naturalness in benchmarks). Rollback: `ALIF_USE_LEGACY_BATCH=1` restores the legacy `generate_sentences_for_words ‚Üí deterministic-validate ‚Üí Haiku-rerank` path. Legacy `generate_material_for_word(model_override="gemini")` was on-demand (fast), `warm_sentence_cache(llm_model="gemini")` for in-session background (defaults to Gemini for speed; cron uses `claude_sonnet` via update_material.py). Dynamic difficulty via `get_sentence_difficulty_params()`. Default needed=2, requests needed+2 to absorb validation failures. `validate_multi_target_sentence()` + `write_multi_target_sentence()` for multi-target sentences ‚Äî the split lets callers run all LLM validation first, then batch the writes, so the SQLite write lock is never held during an LLM call. `warm_sentence_cache()` runs **tier-based lifecycle rotation first** (`rotate_stale_sentences(tier_lookup=...)`), then pre-generates for three gap types: (1) focus cohort words below tier-based target, (2) likely auto-intro candidates with < 3 sentences, (3) **recency-exhausted words** ‚Äî words with enough sentences but ALL shown in last 24h (capped at 20 per warm run). Gap words sorted by tier urgency. **Tier-based lifecycle** replaces fixed cap: `rotate_stale_sentences()` has two retirement paths ‚Äî (1) tier-4 excess (shown sentences immediately, never-shown after 24h), (2) scaffold staleness (all scaffold fully known). Floor uses tier values directly (tier 1‚â•2, tier 2‚â•1, tier 3-4‚â•0) ‚Äî no min_active override. Safety valve cap at 2000 (should never bind). Also triggered as background task after every session load. **Generate-then-write pattern**: all functions close DB before LLM calls (15-30s via Claude CLI), then reopen briefly for writes ‚Äî prevents "database is locked" errors during concurrent access. **NULL lemma_id guard**: all sentence storage paths reject unmapped words (uses `build_comprehensive_lemma_lookup()`). **ALA-LC transliteration override**: both `generate_material_for_word()` and `write_multi_target_sentence()` override LLM transliteration with deterministic ALA-LC from diacritized Arabic via `transliterate_arabic()`. **LLM mapping gate**: when `VERIFY_MAPPINGS_LLM=1`, sentences with LLM-flagged bad mappings are discarded (~10% rejection rate). **Batch mapping verification**: `verify_sentence_mappings(db, sentence_ids)` checks existing sentences in a single batched LLM call (up to 20 sentences per call). Flags wrong mappings, applies corrections via `apply_corrections()` (existing DB lemmas only ‚Äî never auto-creates), retires sentences with unfixable mappings (correct lemma not in DB), stamps `mappings_verified_at`. Gemini ‚Üí Claude Haiku fallback; total failure leaves sentences unverified for retry. Called from `warm_sentence_cache` Phase 4 ‚Äî background catch-up of unverified active sentences, 20 per run. New sentences are pre-stamped at creation via generation-time verification. **Phase 5: Empty-gloss backfill** ‚Äî catches lemmas that slipped through import without English translations. Queries acquiring/known/lapsed/learning lemmas with NULL or empty `gloss_en`, backfills up to 10 per run via LLM batch translation. Self-healing safety net. **Generation backoff**: `record_generation_result()` + `lemmas_on_backoff()` track consecutive 0-result attempts per lemma in `UserLemmaKnowledge.generation_failed_count` / `generation_backoff_until`; after 3 failures a lemma is skipped for 7 days from `words_needing` and `gap_word_ids`, stops chronically-failing lemmas wasting CLI calls on every cron run. Any successful generation clears the counter.

## Stories, Books & Quran
- `quran_service.py` ‚Äî Quranic verse reading mode. Three main functions: (1) `select_verse_cards(db)` ‚Äî picks due review verses + up to 3 new verses (only lemmatized, gated by non-understood backlog < 20), returns word-level data for tap-to-lookup. Populates `gloss_en` for function words from `FUNCTION_WORD_GLOSSES` + `_QURAN_FUNCTION_GLOSSES` (ÿ•ŸäŸëÿßŸÉ forms with "alone" exclusivity, all 14 muqatta'at combinations). (2) `submit_verse_review(db, verse_id, rating)` ‚Äî level-based SRS: "not_yet" ‚Üí immediate, "partially" ‚Üí 2h (level - 1), "got_it" ‚Üí advancing intervals (4h‚Üí12h‚Üí1d‚Üí3d‚Üí7d‚Üí21d‚Üígraduated). (3) `lemmatize_quran_verses(db, limit=20)` ‚Äî lazy lemmatization pipeline: tokenize ‚Üí lookup via `build_lemma_lookup()` + `find_best_db_match()` ‚Üí **ta maftouha fallback** (word-final ÿ™ ‚Üí ÿ© re-lookup for Quranic orthography like ÿ±ÿ≠ŸÖÿ™‚Üíÿ±ÿ≠ŸÖÿ©, ŸÜÿπŸÖÿ™‚ÜíŸÜÿπŸÖÿ©) ‚Üí **hamzat al-wasl fallback** via `_hamzat_wasl_lookup()` (restores dropped initial alef after proclitic stripping, e.g. ÿ®ÿ≥ŸÖ ‚Üí ÿ® + ÿßÿ≥ŸÖ) ‚Üí batch LLM translation for unknowns ‚Üí create Lemma (source="quran") + QuranicVerseWord records. `_create_unknown_quran_lemmas()` gets general Arabic glosses (not Quran-specific theological meanings), extracts consonantal roots in the same LLM call, links/creates Root records, and triggers `enrich_lemmas_batch()` for forms/etymology/transliteration. **Quran lemma promotion**: encountered Quran-only lemmas auto-promote to acquiring when they appear in ‚â•3 distinct verses rated "got_it" (srs_level ‚â• 2). `_maybe_promote_quran_lemmas()` runs after every verse review (`QURAN_PROMOTION_THRESHOLD = 3`). Triggered when <10 lemmatized unseen verses remain. Data: 6236 verses from risan/quran-json CDN (Uthmani tashkeel + Sahih International translation). Tables: `quranic_verses`, `quranic_verse_words`.
- `story_service.py` ‚Äî Generate/import stories. Generation uses Claude Opus with self-correction loop: generate once, then iteratively fix unknown words (up to 3 correction rounds) rather than regenerating from scratch. 100% vocabulary compliance required ‚Äî zero unknown words allowed. `_get_known_words()` includes acquiring words in Leitner box 2+ only (box 1 excluded ‚Äî too fresh). Full vocab list sent during correction rounds for maximum replacement options. POS-grouped vocab in prompts. `_import_unknown_words()` batch-translates unknown story/book words via LLM with positional fallback keying (keys gloss_map by both LLM-returned Arabic and our own surface_bare/lex_norm at each position ‚Äî prevents empty glosses from normalization mismatches). **Empty-gloss guard**: skips creating Lemma entries when LLM translation returns empty English ‚Äî leaves StoryWord unmapped instead of creating glossless vocabulary items. Acquiring words highlighted as reinforcement targets. **4 format types**: standard, long (12-20 sentences), breakdown (splittable sentences for half‚Üífull audio), arabic_explanation (A1 Arabic explanations per sentence, stored in `metadata_json`). Completion creates "encountered" ULK (no FSRS card); only real FSRS review for words with active cards. Suspend/reactivate toggle via `suspend_story()`. **Archive**: `archive_story()` toggles `archived_at` ‚Äî orthogonal to status. Story statuses: active, completed, suspended. Readiness counts `_ACTIVELY_LEARNING_STATES` (acquiring/learning/known/lapsed) ‚Äî not just learning/known. **Story audio**: `generate_story_audio()` produces TTS MP3 with format-aware segment structure (reuses podcast `Seg`/`stitch_podcast` pattern). Voice rotation via `pick_voice_for_story()`. Audio in `data/story-audio/`. **`mark_story_heard()`**: increments `times_heard` on ULK for passive listening credit (no FSRS reviews). **Live count recalculation**: `_recalculate_story_counts()` runs on every `get_story_detail()` call ‚Äî deduplicates by lemma_id, re-checks function word flags (catches words imported before detection was updated), and resolves variant‚Üícanonical knowledge via multi-hop chain following. `_build_knowledge_map()` resolves variant lemma chains to root canonical (A‚ÜíB‚ÜíC uses C's state if more advanced). Function words excluded from unknown_count. `_create_story_words()` checks both surface form and resolved lemma bare form for function word detection. **Cold/warm unknown classification** (2026-03-27): `_classify_unknowns_by_root(db, unknown_ids)` maps unknown lemma IDs to roots, then checks DB-wide for known root siblings. `_compute_cold_warm_counts()` returns `(cold_count, warm_count, reading_readiness_pct)` where `reading_readiness_pct = (known + 0.6√ówarm) / total √ó 100` (0.6 coefficient reflects partial root-family semantic access). Both fields returned in `StoryDetailOut`. `get_pretest_words(db, story_id)` ‚Üí top 5 cold unknowns ranked by token frequency in the story, as `PretestWordOut`. `get_book_page_detail()` for per-page word/sentence breakdown. `_get_book_stats()` computes page-level and story-level progress using `review_log` first-review date to distinguish words genuinely new at import from pre-existing knowledge (resilient to `acquisition_started_at` resets by maintenance scripts). `_verify_new_story_mappings()` commits at function entry (caller already flushed new lemma writes) and after each verification chunk, so the verify LLM calls never run under a held write lock.
- `book_import_service.py` ‚Äî Book import pipeline: per-page OCR ‚Üí per-page LLM cleanup/diacritics/segmentation ‚Üí LLM translation ‚Üí story creation (reuses story_service) ‚Üí sentence extraction (Sentence + SentenceWord records with source="book", page_number tagged). **LLM mapping verification**: when `VERIFY_MAPPINGS_LLM=1`, runs `verify_word_mappings_llm()` on each book sentence ‚Äî bad mappings are nulled out (not discarded, since book sentences can't be regenerated). Creates encountered ULK records with source="book" for new words. Cover metadata extraction via Gemini Vision. Book sentences get 1.3x preference in session builder scoring. Words prioritized via story_bonus + page-based bonus (earlier pages ‚Üí higher priority). CAMeL morphology resolves conjugated forms to existing lemmas. Uploaded images saved to `data/book-uploads/` for retry on failure. Dark image auto-enhancement via Pillow (brightness/contrast boost when mean brightness < 120). Empty OCR results retry with `gemini-2.5-flash-preview` thinking model. Sentences with unmapped tokens kept (lemma_id=None) instead of skipped; StoryWord surface‚Üílemma fallback lookup resolves most unmapped words. `create_book_sentences()` commits per-sentence (not per-book) so the SQLite write lock is released before each iteration's verify LLM call, and partial imports stay durable on crash.

## LLM & NLP
- `llm.py` ‚Äî LLM routing with two paths. **Batch/background**: Claude CLI (free via Max plan) ‚Äî `claude_sonnet` for sentence gen, `claude_haiku` for quality gate + enrichment + hooks. **Latency-sensitive** (user-facing interactive): direct Anthropic API via litellm (`model_override="anthropic"` ‚Üí `claude-haiku-4-5`) ‚Äî CLI subprocess adds ~2-3s startup, unacceptable for real-time UX. Current direct-API paths: `/api/chat/ask`. `_generate_via_claude_cli()` shells out to `claude -p` with `--output-format json`. MODELS list for API fallback: openai (GPT-5.2), anthropic (Haiku), opus. JSON mode, markdown fence stripping, model_override. `format_known_words_by_pos()` for POS-grouped vocabulary. `generate_sentences_multi_target()` for multi-word sentences. `review_sentences_quality()` ‚Äî Claude Haiku quality gate (grammar + translation accuracy, fail-closed).
- `claude_code.py` ‚Äî Claude Code CLI (`claude -p`) wrapper. Two modes: (1) `generate_structured()` ‚Äî no tools, `--json-schema` for single-turn output; (2) `generate_with_tools()` ‚Äî `--tools "Read,Bash"` + `--dangerously-skip-permissions` + `--add-dir` for multi-turn agentic sessions where Claude reads vocab files and runs validation scripts (timeout: 240s, budget cap: $0.50). `dump_vocabulary_for_claude()` exports full learner vocabulary to prompt file (with "CURRENTLY LEARNING" section for acquiring words) + lookup TSV. Callers fall back to litellm when unavailable.
- `morphology.py` ‚Äî CAMeL Tools analyzer. Hamza normalized at comparison time only (preserved in storage). Falls back to stub if not installed.
- `transliteration.py` ‚Äî Deterministic Arabic‚ÜíALA-LC romanization from diacritized text. Handles long vowels, shadda, hamza carriers, alif madda/wasla, sun letter assimilation, tƒÅ æ marb≈´·π≠a, nisba ending. **Uthmani diacritics**: recognizes U+06E1 (small high dotless head of khaa / Uthmani sukun), U+06DF (small high rounded zero), U+06E2 (small high meem) so Quranic text transliterates correctly. **Long-vowel inference for partially-vocalized text** (fixed 2026-05-04): bare ya/waw following a vowelless consonant infers long ƒ´/≈´ (e.g. `ÿ≠ŸéÿØŸäŸÇÿ©` ‚Üí `·∏•adƒ´qa`, `ÿ•Ÿäÿ¨ÿßÿ±` ‚Üí `ƒ´jƒÅr`), mirroring the existing bare-alif ‚Üí long ƒÅ logic. Word-initial hamza-carriers (ÿ• ÿß ÿ£ Ÿ±) handle long ƒ´/≈´ the same way. **Consonant-glide disambiguation**: a ya/waw is treated as a consonant ‚Äî not a long-vowel marker ‚Äî when (a) it carries its own short vowel (e.g. `ÿ≥ŸêŸäŸéÿßÿ≥Ÿéÿ©` ‚Üí `siyƒÅsa`, not `sƒ´ƒÅsa`) or (b) it's immediately followed by alif/maqsura (e.g. `ÿ≠ŸéÿßŸÑŸêŸäÿßŸã` ‚Üí `·∏•ƒÅliyƒÅ`, not `·∏•ƒÅlƒ´ƒÅ`), since Arabic phonotactics disallow two adjacent long vowels. `transliterate_lemma()` for dictionary form (strips tanwƒ´n + case vowels). `transliterate_forms()` iterates forms_json values and produces parallel ALA-LC transliterations (skips metadata keys like "gender", "verb_form").
- `variant_detection.py` ‚Äî Three-layer variant detection: (1) CAMeL candidates with root_id validation (rejects different-root pairs), (2) Gemini Flash LLM confirmation with VariantDecision cache, (3) display fix in sentence_selector uses original lemma_id. Used by ALL import paths. Graceful fallback if LLM unavailable.
- `confusion_service.py` ‚Äî Rule-based confusion analysis for "did not recognize" (yellow) words. Four analysis types: (1) **morphological** ‚Äî decomposes surface form into prefix clitics + stem + suffix clitics using PROCLITICS/ENCLITICS lists, matches stem against lemma and forms_json entries; (2) **visual** ‚Äî finds similar-looking words in user's vocabulary (including encountered) via edit distance (‚â§2) + rasm skeleton distance (dots-removed comparison). Rasm groups map letters differing only by dots to same skeleton (ÿ®/ÿ™/ÿ´/ŸÜ ‚Üí same base); (3) **phonetic** ‚Äî finds words that sound similar to learners but look different via `PHONETIC_MAP` (emphatic‚Üíplain: ÿµ‚Üíÿ≥, ÿ∂‚ÜíÿØ, ÿ∑‚Üíÿ™, ÿ∏‚Üíÿ∞; pharyngeal: ÿ≠‚ÜíŸá, ÿπ‚Üíÿß; interdental: ÿ´‚Üíÿ≥, ÿ∞‚Üíÿ≤; uvular: ÿ∫‚ÜíÿÆ). Catches confusions like ÿ≥ÿ®ÿπ‚Üîÿµÿ®ÿßÿ≠. Only surfaces words NOT already in visual results (edit distance >2); (4) **prefix disambiguation** ‚Äî when a word starts with Ÿà/ŸÅ/ÿ®/ŸÑ/ŸÉ, hints whether it's a proclitic prefix or part of the root (uses `lemma.root` relationship). All rule-based, no LLM, <50ms. Endpoint: `GET /api/review/confusion-help/{lemma_id}?surface_form=...`.
- `grammar_service.py` ‚Äî 49 features, 8 tiers. Comfort score: 60% log-exposure + 40% accuracy, decayed by recency.
- `grammar_tagger.py` ‚Äî LLM-based grammar feature tagging.
- `grammar_lesson_service.py` ‚Äî LLM-generated grammar lessons, cached in DB.
- `import_quality.py` ‚Äî LLM batch classify + filter for word imports. `classify_lemmas()` categorizes each word (standard/proper_name/onomatopoeia/junk) and rejects junk. `filter_useful_lemmas()` is a backward-compat wrapper. Used by OCR, story import, and book import paths.
- `lemma_quality.py` ‚Äî Centralized post-creation quality gate. `run_quality_gates(db, lemma_ids)` ‚Äî single pipeline all import paths call after creating Lemma records: (1) finalize ‚Äî clean bare forms, assign frequency ranks, flag dupes, (2) variant detection ‚Äî LLM + definite + mark, (3) enrichment ‚Äî forms/etymology/transliteration/roots/grammar/examples (background thread by default), (4) stamp `gates_completed_at`. Session builder rejects lemmas with NULL `gates_completed_at`. Cron catch-all in `update_material.py` Step G2.

## Audio & Enrichment
- `tts.py` ‚Äî ElevenLabs REST, eleven_multilingual_v2, PVC clone of @roots_of_knowledge, speed 0.7. Learner pauses. SHA256 cache. Voice pool (`ARABIC_VOICE_POOL`, 3 voices) with `pick_voice_for_story(story_id)` for deterministic rotation. Story audio dir: `data/story-audio/`.

- `memory_hooks.py` ‚Äî LLM-generated memory aids (mnemonic, cognates, collocations, usage context, fun fact). **Generated on first failure, not on introduction** ‚Äî avoids wasting processing on already-known words. Idempotent. **All generation uses overgenerate-and-rank**: generates 3 candidate mnemonics with different keywords, self-evaluates each on sound match/interaction/extraction (1-5 scale), picks best. Uses Claude Sonnet CLI (free). Targets 13 learner languages for cognate search. Prompt uses keyword mnemonic method (Atkinson & Raugh 1975). Enforces interactive imagery (keyword + meaning must interact), self-reference ("you"), meaning-as-action, keywords in CAPS. Abstract words get consequence-based or verbal mnemonics. **Trigger points** (all via background thread): (1) first failure (rating ‚â§ 2) when no hooks exist ‚Üí `generate_memory_hooks()`, (2) FSRS lapse with existing hooks ‚Üí `regenerate_memory_hooks_premium()` (feeds old mnemonic as negative example), (3) acquisition box demotion from 2+ ‚Üí 1 with existing hooks ‚Üí same premium regeneration.
- `root_enrichment.py` ‚Äî LLM-generated root enrichment (etymology story, cultural significance, literary examples, fun facts, related roots). Uses Claude Sonnet CLI (free). `generate_root_enrichment(root_id)` ‚Äî idempotent, skips if enrichment exists. `maybe_enrich_root(root_id, db)` ‚Äî checks if root has 2+ studied lemmas and no enrichment, triggers background thread. Hooked into `start_acquisition()`.
- `pattern_enrichment.py` ‚Äî LLM-generated pattern enrichment (explanation, how to recognize, semantic fields, example derivations, register notes, fun facts, related patterns). Uses Claude Haiku CLI (free). `generate_pattern_enrichment(wazn)` ‚Äî idempotent, creates/updates PatternInfo row. `maybe_enrich_pattern(wazn, db)` ‚Äî checks if pattern has 2+ studied words and no enrichment, triggers background thread. Hooked into `start_acquisition()`.
- `lemma_enrichment.py` ‚Äî Batch enrichment for newly created Lemma records. `enrich_lemmas_batch(lemma_ids)` populates: (1) transliteration_ala_lc (deterministic), (2) forms_json via Claude Haiku, (3) etymology_json via Claude Haiku (batched), (4) root_id ‚Äî extracts consonantal roots via LLM, creates Root records as needed + backfills root meanings, (5) grammar_features_json via `tag_lemma_grammar()`, (6) example_ar/example_en via LLM (batched).
