# Frontend Files

All frontend in `frontend/`. The Expo app serves **two languages** (Arabic via the Alif backend, Modern Greek via the Polyglot backend); the user switches via the Globe tab. See *App shell & language routing* below for how the two sides stay isolated.

## App shell & language routing
- `app/_layout.tsx` — Top-level Tabs container. Tab visibility (`href: null` vs `undefined`) is driven by the active language from `useLanguage()`. Two guards keep the tab bar and the active route from disagreeing: (1) renders an `ActivityIndicator` until the language context's `ready` flag flips (so Expo Router can't lock in an Arabic-side default route before `AsyncStorage` resolves), and (2) a `useEffect` watches `pathname` + `language` and `router.replace()`s to the active language's home whenever they disagree. The cold-start bug it prevents: stored language `"el"` briefly resolved URL `/` to the Arabic Reading screen, leaving Arabic content under a Greek tab bar. **`backBehavior="history"`** is set on the `<Tabs>`: every detail page (`word/[id]`, `polyglot-lemma/[id]`, …) is a flat sibling tab, not a `<Stack>` screen, so the bottom-tab default `backBehavior="firstRoute"` made `router.back()` jump to the *first* tab (`index` = Arabic Reading) instead of the tab you opened the detail from. Alif tolerated it (its detail screens open from `index`); Polyglot opened philology from `polyglot-review`, so back landed on Arabic `index` → the language-sync `router.replace` then bounced to `/polyglot` (Greek Reading) — never Review. `history` returns back to the previously focused tab. **Greek tab order**: Review (`polyglot-review`) is the first Greek tab, then Reading, then Stats. **Language popover**: the Globe tab's `tabPress` is intercepted (`e.preventDefault()`) to toggle a small popover (`LANGUAGE_OPTIONS` + `pickerBackdrop`/`pickerCard` styles) anchored over the globe, instead of navigating to the full `languages` screen. Each row shows only the native script (`العربية`, `Ελληνικά`, `Lingua Latina`) — no English subtitle. Picking a language **only** calls `setLanguage()` and lets the language-sync `useEffect` above do the navigating. It must NOT `router.replace()` synchronously here: tab visibility is `href`-driven, so the destination tab (e.g. `polyglot-review`) is still `href: null`/unregistered until the *next* render reflects the new language — a synchronous replace raced that flip and threw *“The action 'REPLACE' … was not handled by any navigator. Do you have a route named 'polyglot-review'?”*. Deferring to the effect means the tab is registered before the redirect runs (same path cold-start already relies on).
- `app/languages.tsx` — Full-screen language-picker **fallback**. The live picker is now the popover in `_layout.tsx` (the globe's `tabPress` is intercepted), so this screen is no longer reached in normal use; retained as a route fallback (e.g. a direct `/languages` web deep link) and to keep the globe tab registered. Still functional: tapping a language calls `setLanguage()` and then navigates to that language's home — but **deferred**, because `/languages` is a `shared` route that `_layout`'s language-sync effect deliberately skips. It stashes the target path in a ref and `router.replace()`s from a `useEffect` keyed on `language`, i.e. after `_layout` has re-rendered the destination tab as registered (avoids the same REPLACE-not-handled race as the popover). Shared route.

**Isolation rule**: URL prefix is the source of truth. `/languages` is shared; anything under `/polyglot`, `/polyglot/*`, or `/polyglot-*` is Greek; everything else is Arabic. New Greek screens just need to follow the `polyglot-*` filename convention and they inherit isolation automatically. `lib/__tests__/language-context.test.ts` fails CI if a new file appears in `app/` that hasn't been explicitly classified — that's the guardrail. The Arabic and Greek API clients (`lib/api.ts` vs `lib/polyglot-api.ts`) and offline sync queue (Arabic-only) are already cleanly separated; the routing isolation completes the story.

## Screens (app/)
- `app/index.tsx` — Review screen: sentence-only (no word-only fallback), reading + listening, word lookup, word marking, back/undo, wrap-up mini-quiz (acquiring + missed words), session word tracking, story source badges on intro cards, reintro cards (retention-optimized layout with pattern/root/etymology/mnemonic [mnemonic hidden since 2026-05-22 via `SHOW_MNEMONIC_HOOKS`] + Continue button — **verb form derivation bridge** in Pattern block for Form II–X), experiment intro cards (interleaved among sentences via `buildInterleavedSession()` — event-driven: warm-up sentences first, then each intro card emitted immediately before the first sentence containing its lemma; the user never sees a word in a sentence and gets the intro card after — **derivation bridge** in Root Family section + form labels on siblings), session end card (journey timeline with graduated/advanced/slipped words, pipeline bar with deltas, response speed comparison, today stats with reviewed count + 7-day retention %, root coverage — fetches lightweight session-end endpoint concurrently with sync flush (not blocked by it) — single call replaces previous 3-call pattern). When the completion screen appears, it drops any unreviewed cached tail for that `session_id`; the Next Session button forces a fresh server session so abandoned tails cannot reappear as one-card "new" sessions or pollute session-end stats.
- `app/learn.tsx` — Learn mode: 5-candidate pick → done (no quiz). Redesigned info-dense card with hero section (Arabic word, gloss, transliteration, flow chips for POS/CEFR/freq/root/pattern) + scrollable info sections below (memory hook [mnemonic hidden since 2026-05-22], etymology, cognates, root family, pattern examples, usage context, fun fact/cultural note). Root and pattern chips are tappable → navigate to `/root/{id}` and `/pattern/{wazn}`. **Verb form derivation bridge**: for Form II–X verbs, the Root Family section shows "causative/intensive of [Form I: arabic — gloss]" using `VERB_FORM_RELATION` map + `root_family` sibling data. No play button (word-level TTS disabled). Story source badge for story words.
- `app/explore.tsx` — Explore tab with three sub-tabs (Words/Roots/Patterns). Words sub-tab: grid, category tabs (Vocab/Function/Names), smart filters (Leeches/Struggling/Recent/Solid/Next Up/Acquiring/Encountered), sparklines, search. Roots sub-tab: FlatList of root cards (Arabic letters, core meaning, coverage bar), search, tap → `/root/{id}`. Patterns sub-tab: FlatList of pattern cards (name, meaning, coverage bar), search, tap → `/pattern/{wazn}`.
- `app/words.tsx` — Redirect to /explore (legacy URL support)
- `app/stats.tsx` — Analytics dashboard organized into 5 sections: **Today** (hero card with comprehension bar, reviewed count + 7-day retention %, graduated pills, new words by source, calibration, today's transitions), **Vocabulary** (unified card: hero known count with weekly/daily trend, proportional flow strip across all stages, flow rate annotations, detail cells for acquiring+learning, lapsed+retention+roots chips, textbook comparison bars (Al-Kitaab Part 1, Madinah Book 1), Quran progress (verses studied/mastered/graduated), milestone checklist (1000 words, 500 roots, Madinah >70%, Al-Fatihah, etc.), acquisition pipeline with pressure bars per Leitner box), **Progress** (14-day activity chart with words_learned overlay, learning pace 7d+30d with truncated decimals, pages read all-time + weekly, transitions 7d+30d, retention 7d+30d, comprehension 7d+30d), **Sessions** (recent sessions with mini comprehension bars), **Deep Dive** (vocabulary health/stability distribution, struggling words, root progress, insights card — untapped root, forgetting forecast, graduation stats)
- `app/story/[id].tsx` — Story reader with tap-to-lookup, WordInfoCard for lookups, ActionMenu in header bar (Ask AI, suspend story, mark as heard, play/stop audio). Audio player via expo-av when `audio_filename` exists (play button in header bar + action menu). All navigation (back, complete, suspend) goes to `/stories` via `router.replace`. **Reading Readiness banner** (2026-03-27): shown for active stories with cold unknowns — displays `X% ready · N new · M familiar root` and a "Preview" button. Tapping Preview opens the **pretesting flash modal**: cycles through top 5 cold unknowns (from `/api/stories/{id}/pretest-words`), shows Arabic word for 2s then reveals gloss + root chip. Failed retrieval attempt primes encoding of the subsequently revealed answer (Richland et al. 2009, d=1.1).
- `app/stories.tsx` — Story list with generate + import + book import, grouped sections (Active/Suspended/Completed/Archived — last two collapsed), archive/unarchive toggle per story, format badge (Long/Breakdown/Arabic Only), audio icon, format picker in generate modal (standard/long/breakdown/arabic_explanation), suspend all, suspend/reactivate toggle per story, book source badges, completion prediction (~Xd until ready), clickable per-page readiness pills for book imports (learned/new format, green when all page words acquiring; navigate to book-page detail), book footer shows "X/Y new words learning" (deduplicated, only words unknown at import)
- `app/book-page.tsx` — Book page detail: words (new vs already known) with status pills, sentences with seen/unseen indicators. Navigable from page pills on story list.
- `app/scanner.tsx` — Textbook page OCR scanner. Scanned words are presented as high-priority learning candidates, not as "preserve known" cards; the frontend no longer exposes the legacy preserve toggle. **In-app burst camera** (2026-05-14): tapping "Camera" opens a full-screen `CameraView` (expo-camera) with a shutter button — captures stay in a horizontal thumbnail strip and only get added to the upload list when you tap Done; Cancel discards. On each shutter: 240ms white flash overlay + thumbnail strip auto-scrolls to the newest capture (so feedback stays visible after the strip fills). `capturing` guard prevents double-shutter. "Gallery" still uses `ImagePicker.launchImageLibraryAsync` for picking from photos. Source HTMLs for the home-screen icon are committed under `frontend/assets/sources/` (variant H "Framed" from the March 2026 design pass — regenerate via the `icon-generation` skill).
- `app/snap.tsx` — **Snap-to-read** tab (Arabic only). Photograph any Arabic page (camera or library) → one synchronous `POST /api/discover/snap` returns a faithful English translation + the top ~5 unknown words. Renders the translation card, a collapsible original-Arabic block, and per-word chips (vocalized headword, gloss, POS + register/dialect badges) each with an Add button (→ `addDiscoveredWord`, idempotent: shows "Known" when `already_known`). "Add all" walks the list. For interactive in-the-moment reading help, distinct from the batch `scanner.tsx`/`book-import.tsx` flows.
- `app/book-import.tsx` — Book import: photograph cover + content pages → reading goal with sentence extraction
- `app/more.tsx` — More tab: Scanner, Chats, New Words, Activity Log
- `app/word/[id].tsx` — Word detail: forms, root family, pattern, etymology, memory hooks (mnemonic hidden since 2026-05-22; cognates/collocations/usage/fun-fact still shown), review history, sentence stats. Root section shows whenever `word.root` exists (even without family members), Pattern section shows whenever `word.wazn` exists (even without pattern examples). Root title links to `/root/{root_id}`, Pattern title links to `/pattern/{wazn}`. Actions (postpone/suspend/flag) in "..." overflow menu in header bar.
- `app/root/[id].tsx` — Root detail: 52pt Arabic root letters, core meaning, enrichment section (etymology story, cultural significance, literary examples, fun facts, related roots as pills), derivation tree grouped by pattern. Words tappable → `/word/{id}`, pattern headers tappable → `/pattern/{wazn}`.
- `app/pattern/[id].tsx` — Pattern detail: pattern name + meaning, enrichment section (explanation, how to recognize, semantic fields, example derivations, register notes, fun facts, related patterns as pills), word list sorted by knowledge state. Words tappable → `/word/{id}`.
- `app/chats.tsx` — AI chat conversations
- `app/listening.tsx` — Dedicated listening mode
- `app/review-lab.tsx` — Hidden route for testing review UI variants

## Components (lib/)
- `lib/review/ActionMenu.tsx` — "⋯" menu: Ask AI (auto-explain on review), Suspend, Flag. Supports `extraActions` for screen-specific actions, injectable Ask AI / flag / suspend clients for sibling backends such as polyglot, and `sentenceReportLabel` for screens that want a single "Report sentence" action instead of Alif's expanded sentence-flag submenu. Computes auto-explain prompt eagerly at open time.
- `lib/review/WordInfoCard.tsx` — Word info panel for review. Always shows full info (no root gate). Pattern link clickable → `/pattern/{wazn}`. Root line clickable → `/root/{id}` (via `onNavigateToRoot` prop). Root siblings tappable → `/word/{id}` (via `onNavigateToDetail`). Only known/learning root siblings shown. Prev/next arrows navigate tapped word history. Shows surface form transliteration for conjugated forms, full etymology (derivation, loanwords, cultural note), and memory hooks (mnemonic hidden since 2026-05-22 via `SHOW_MNEMONIC_HOOKS`; cognates, collocations, usage context, fun fact still shown). **Confusion analysis**: when a word is marked "did not recognize" (yellow), shows: (1) inline-highlighted confusable words — matching letters dimmed (25% opacity), differing letters in orange, with `splitArabicGraphemes()` for proper diacritic grouping; (2) teaching hints from backend `match_reason`/`matched_form` plus "dots only" pill for rasm_distance=0; (3) color band morphological decomposition — surface form with each morpheme colored (purple/orange/blue/green) + legend. `HighlightedArabic` component handles the inline diff rendering. Scrollable when content overflows (maxHeight 200).
- `lib/review/SentenceInfoModal.tsx` — Debug modal: sentence ID, source, review history, per-word FSRS difficulty/stability. **Selection reasoning**: when opened from review, shows why the sentence was chosen (scheduled review / acquisition repeat / on-demand / auto-intro fill), primary word state, selection score + pick order, and per-factor score breakdown
- `lib/review/ConfusionPicker.tsx` (2026-05-27) — Capture user-reported confusions on yellow ("did not recognize") taps. Default state is a small collapsed "Confused with another word?" link below the WordInfoCard. Tap to expand into up to 5 similar/phonetic candidate chips from `confusionData` + a free-text input ("…or type in English"). Picking a chip OR submitting non-empty text calls `onSave` with a `ConfusionCaptureIn` object (`capture_method: 'suggested_pick' | 'free_text'`); the text input and chip rows mutually exclude — picking dims the input, typing dims the chips. Captures collected per `lookupLemmaId` and submitted alongside the sentence review (`confusion_captures` field in `submitSentenceReview`). State cleared on card advance, mode change, or unmarking a yellow word.
- `lib/AskAI.tsx` — AI chat modal (used in ActionMenu). Auto-sends combined explain prompt on open from review screen (sentence understanding + marked word breakdowns + lemma accuracy check + translation accuracy check). Shows auto-sent message as "Explaining this sentence..." label. Follow-up questions supported. Flag button after response. **Error resilience**: component-level 35s timeout safety net (independent of fetch AbortController), 8s slow warning, retry button on errors, friendly error messages.
- `lib/MarkdownMessage.tsx` — Markdown renderer for chat/AI responses
- `lib/WordCardComponents.tsx` — Reusable word display (posLabel, FormsRow, FormsStrip with per-form transliteration, PatternExamples, PlayButton)

## Infrastructure (lib/)
- `lib/api.ts` — API client with typed interfaces for all endpoints. Offline-first: `introduceWord()`, `submitReintroResult()`, `acknowledgeExperimentIntro()`, `introduceGrammarFeature()` all enqueue to sync queue with optimistic returns. Auto-prefetch: `backgroundPrefetch()` silently caches 2 more sessions after every session load (fresh, cached, or fallback). Session load cascade: online cache with at least 5 remaining sentence items → stale cache only when offline → fetch with 12s timeout → stale cache fallback on fetch failure.
- `lib/language-context.tsx` — React Context for the active language. Persists selection to `AsyncStorage` (key `@app:active-language`), exposes `{ language, setLanguage, ready }`. Re-exports `AppLanguage`, `routeLanguage`, and `homePathFor` from `language-routes.ts` for ergonomic consumers.
- `lib/language-routes.ts` — Pure helpers (no React) for the Arabic ↔ Greek isolation: `routeLanguage(pathname)` classifies a URL as `"ar" | "el" | "shared"`, `homePathFor(lang)` returns the entry route. Kept React-free so the manifest test can import it without dragging Expo into the test runtime.
- `lib/types.ts` — TypeScript interfaces
- `lib/offline-store.ts` — AsyncStorage session cache (versioned keys, 30-min staleness TTL, stale allowed when offline) + reviewed tracking with auto-pruning (caps at 1000 entries, prunes to keys matching cached sessions). `getCachedSession()` can require a minimum remaining-card count so online "new session" loads skip depleted fragments; `dropCachedSession()` removes an abandoned session tail after wrap-up. Session cache writes are serialized to avoid cache/drop races. Background refresh via `fetchFreshSession()` for in-session staleness (15-min gap detection via AppState). Word lookup cache: versioned key (`v2`), 24h TTL per entry, `allowStale` fallback for offline use.
- `lib/sync-queue.ts` — Offline-first queue for all mutable actions. Entry types: `sentence` (batch via /api/review/sync), `story_complete`, `introduce_word`, `reintro_result`, `experiment_intro_ack`, `grammar_intro` (individual POST). Max 8 retry attempts before dropping. Queue lock serializes operations. Flush triggers: app resume, network-online event, after each review submission.
- `lib/theme.ts` — Dark theme, semantic colors, dual Arabic font families (Scheherazade New + Amiri), `arabicFontForSentence()` for 50/50 font mixing by sentence_id, `ltr()` helper (prepends U+200E LRM for mixed BiDi text). **BiDi convention**: pure Arabic text gets `writingDirection: "rtl"`, mixed Arabic+English explanatory text gets `writingDirection: "ltr"` + `ltr()` wrapper — the LRM is needed because iOS determines paragraph direction from the first strong character, overriding the style property.
- `lib/net-status.ts` — Network status singleton + useNetStatus hook
- `lib/sync-events.ts` — Event emitter for sync notifications
- `lib/frequency.ts` — Frequency band + CEFR color utilities
- `lib/grammar-particles.ts` — Rich grammar info for 12 core Arabic particles (في، من، على, etc.), displayed via GrammarParticleView in WordInfoCard
- `lib/topic-labels.ts` — Human-readable labels + icons for 20 thematic domains
- `lib/mock-data.ts` — Mock words, stats, learn candidates for testing
- `lib/__tests__/` — Jest tests for sync, store, smart-filters, API, typechecks
